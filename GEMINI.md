# Доменный светофор.AI - Руководство для AI-агентов (GEMINI)

Привет, будущий ИИ-агент! Этот документ создан для того, чтобы помочь тебе быстро вникнуть в архитектуру, стек и правила работы с проектом "Доменный светофор.AI".

## 1. О проекте
"Доменный светофор.AI" — это сервис (и расширение), который анализирует домены на предмет безопасности, оценивает риски и выдает вердикт: Low (безопасно), Medium (подозрительно), High (опасно). Проект имеет образовательную направленность, помогая пользователям принимать решение перед переходом по подозрительным ссылкам.

## 2. Стек технологий
*   **Фронтенд:** React 18, TypeScript, Vite, Tailwind CSS (с кастомной конфигурацией для анимаций и стилей), Framer Motion (для сложных анимаций и переходов), Radix UI / Shadcn (для базовых компонентов).
*   **Бэкенд:** Express.js (`server/openrouter-proxy.mjs`). Работает как прокси к OpenRouter AI API.
*   **Кэширование/База Данных:** Локальное кэширование или Upstash Redis.
*   **Роутинг:** Кастомный hash-роутер (`src/lib/site-router.ts`). Обрати внимание, что стандартный `react-router-dom` **не используется**.

## 3. Структура проекта
*   `src/pages/` - основные страницы (Home, Analyzer, Admin, Safety, Method, BrandKit, Changelog и др.). Роутинг идет через `App.tsx` и `site-router.ts`.
*   `src/components/` - UI-компоненты (SiteHeader, SiteFooter, UI-kit).
*   `src/lib/` - утилиты, API-хэлперы, контент (`site-content.ts`), настройка роутинга.
*   `api/` - серверные функции Vercel (серверлес).
*   `server/` - локальный бэкенд для разработки.
*   `extension/` - файлы для браузерного расширения.

## 4. Сборка и Деплой (Vercel)
Проект разворачивается на **Vercel**. Очень важно, чтобы билд проходил без ошибок TypeScript:
1.  Сборка запускается командой `npm run build` (под капотом: `tsc -b && vite build`).
2.  Любая ошибка типов (даже в тестовых файлах, если они не исключены) сломает деплой.
3.  **Перед любым Pull Request или submit**, обязательно проверяй компиляцию: `npm run build`!

## 5. Роутинг (Важно!)
В приложении используется кастомный Hash-роутинг:
*   Пути определяются в `src/lib/site-router.ts`.
*   Если ты добавляешь новую страницу, тебе нужно:
    1.  Добавить путь в `sitePaths` в `src/lib/site-router.ts`.
    2.  Добавить импорт и условие в `renderPage` внутри `src/App.tsx`.
    3.  При необходимости обновить `routeTitles` в `App.tsx`.

## 6. Тестирование
*   Используется встроенный `node:test`.
*   Запуск тестов: `npm run test` (`node --experimental-strip-types --test src/lib/*.test.ts`).
*   Если пишешь тесты для React, нужно мокать DOM (`jsdom`).

## 7. Админка
*   Админка доступна по пути `/#/admin`.
*   Она работает с кэшем запросов и позволяет исправлять ответы AI.
*   Доступ к API админки требует токена (настраивается в бэкенде).

## 8. Дизайн и Стилизация
*   Используй классы Tailwind.
*   Основные цвета привязаны к `background` и `foreground` в CSS переменных (см. `src/index.css`).
*   Для анимаций используется `framer-motion` (компонент `motion`).

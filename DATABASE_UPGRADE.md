# Обновление системы базы данных

## Что изменилось

Система базы данных была полностью переписана для улучшения производительности и надежности.

### Старая система (v1)
- ❌ Дублирование записей для одного домена
- ❌ Огромные вложенные объекты (67KB для 9 записей)
- ❌ Линейный поиск по всем записям
- ❌ Отсутствие индексов
- ❌ Ручное управление сохранением

### Новая система (v2)
- ✅ Одна запись на домен (автоматическая дедупликация)
- ✅ Компактная структура (35KB для 9 записей - экономия 48%)
- ✅ Быстрый поиск по индексу host
- ✅ Автоматическое TTL (7 дней)
- ✅ Автосохранение с debounce
- ✅ Автоматические бэкапы
- ✅ Статистика и мониторинг

## Новые модули

### `server/db-manager.mjs`
Основной модуль управления базой данных.

**Основные методы:**
- `getByHost(host)` - получить запись по домену
- `set(host, data)` - сохранить/обновить запись
- `update(host, updates)` - обновить существующую запись
- `deleteByHost(host)` - удалить запись
- `getRecent(limit)` - получить последние записи
- `search(query)` - поиск по домену/описанию
- `cleanup()` - очистка устаревших записей
- `getStats()` - статистика базы
- `backup()` - создать резервную копию

**Пример использования:**
```javascript
import { getDatabase } from './server/db-manager.mjs';

const db = getDatabase();

// Сохранить запись
db.set('example.com', {
  verdict: 'low',
  score: 10,
  summary: 'Домен выглядит безопасно',
  reasons: [...],
  actions: [...]
});

// Получить запись
const record = db.getByHost('example.com');

// Статистика
const stats = db.getStats();
console.log(`Активных записей: ${stats.active}`);
```

### `server/db-migrate.mjs`
Модуль миграции данных из старого формата в новый.

**Функции:**
- `migrateOldDatabase()` - ручная миграция
- `autoMigrate()` - автоматическая миграция при старте

**Что делает миграция:**
1. Читает старую базу `threat-db.json`
2. Группирует записи по домену
3. Берет только самую свежую запись для каждого домена
4. Сохраняет в новый формат `threat-db-v2.json`
5. Создает бэкап старой базы

## Автоматическая миграция

При первом запуске сервера миграция происходит автоматически:

```
[Migration] Auto-migration triggered
[Migration] Starting migration from v1 to v2...
[Migration] Old database backed up to: threat-db.backup-1774891473966.json
[Migration] Complete: 9 migrated, 0 skipped, 0 errors
[DB] Saved 9 records
```

## Ручная миграция

Если нужно запустить миграцию вручную:

```bash
node server/db-migrate.mjs
```

## Совместимость

Новая система полностью совместима со старым API:
- Все существующие endpoints работают без изменений
- Redis кэш продолжает работать как раньше
- На Vercel всё работает без изменений

## Производительность

### Скорость поиска
- **Старая система:** O(n) - линейный поиск по всем записям
- **Новая система:** O(1) - прямой доступ через индекс

### Размер базы
- **Старая система:** 67KB для 9 записей (7.4KB на запись)
- **Новая система:** 35KB для 9 записей (3.9KB на запись)
- **Экономия:** 48% меньше места

### Дедупликация
- **Старая система:** Множественные записи для одного домена
- **Новая система:** Только одна актуальная запись

## Автоматическая очистка

База автоматически очищается от устаревших записей:
- TTL по умолчанию: 7 дней
- Автоочистка: каждый час
- Записи старше TTL удаляются автоматически

## Резервное копирование

Автоматические бэкапы создаются:
- При вызове `db.backup()`
- Перед миграцией
- Перед полной очисткой `db.clear()`

Хранятся последние 5 бэкапов в `server/backups/`.

## Откат на старую версию

Если нужно вернуться к старой системе:

1. Остановите сервер
2. Удалите `threat-db-v2.json`
3. Восстановите из бэкапа:
   ```bash
   Copy-Item server/threat-db.backup-*.json server/threat-db.json
   ```
4. Откатите изменения в `server/openrouter-proxy.mjs`

## Проблемы и решения

### База не мигрирует автоматически
- Проверьте наличие `threat-db.json`
- Запустите миграцию вручную: `node server/db-migrate.mjs`

### Записи не сохраняются
- Проверьте права на запись в директорию `server/`
- Проверьте логи: `[DB] Saved X records`

### Большой размер базы
- Запустите очистку: `db.cleanup()`
- Уменьшите TTL в настройках

## Конфигурация

Настройки в `server/openrouter-proxy.mjs`:

```javascript
const localDb = getDatabase({
  ttlMs: 7 * 24 * 60 * 60 * 1000,  // TTL записей (7 дней)
  autoSave: true,                   // Автосохранение
  saveDebounceMs: 2000              // Задержка перед сохранением
});
```

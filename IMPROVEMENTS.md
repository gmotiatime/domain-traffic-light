- Более строго следует правилам не смягчения вердикта

### 3. Новый API endpoint для выгрузки данных из БД

#### GET /api/lookup

Новый endpoint для получения кэшированных результатов анализа по ссылке.

**Параметры запроса:**
- `url` или `link` - URL или домен для поиска в кэше

**Примеры использования:**

```bash
# Поиск по домену
curl "https://your-domain.com/api/lookup?url=google.com"

# Поиск по полной ссылке
curl "https://your-domain.com/api/lookup?link=https://example.com/path"
```

**Успешный ответ (200):**
```json
{
  "ok": true,
  "host": "google.com",
  "cached": true,
  "cachedAt": 1774880581307,
  "updatedAt": 1774880581307,
  "result": {
    "verdict": "low",
    "score": 0,
    "verdictLabel": "Низкий риск",
    "summary": "Сильных тревожных признаков не найдено...",
    "reasons": [...],
    "actions": [...],
    "breakdown": {...},
    "analyzedAt": "2026-03-30T14:23:01.307Z"
  },
  "model": "llama-3.3-70b-versatile",
  "source": "groq"
}
```

**Ошибка - домен не найден (404):**
```json
{
  "error": "Данные для этого домена не найдены в кэше.",
  "host": "unknown-domain.com",
  "suggestion": "Выполните анализ через /api/analyze для получения данных."
}
```

**Ошибка - некорректный ввод (400):**
```json
{
  "error": "Параметр url или link обязателен."
}
```

#### Особенности работы

1. **Поиск по хосту**: Endpoint извлекает hostname из URL и ищет его в кэше
2. **Нормализация**: Автоматически добавляет https:// если протокол не указан
3. **Дедупликация**: Возвращает только самую свежую запись для каждого хоста
4. **Поддержка Redis и локального кэша**: Работает как с Vercel Redis, так и с локальным JSON-файлом

#### Интеграция с существующей системой

Endpoint использует те же функции нормализации и кэширования, что и основной `/api/analyze`:
- `normalizeInput()` - валидация и нормализация URL
- `getRawCacheRecordByHost()` - поиск в кэше по хосту
- Поддержка legacy cache prefixes для обратной совместимости

### 4. Vercel Serverless Function

Создан отдельный файл `api/lookup.mjs` для Vercel, который:
- Работает как serverless function
- Использует Upstash Redis напрямую
- Поддерживает те же параметры и формат ответа
- Автоматически деплоится на Vercel

## Использование в браузерном расширении

Пример интеграции в расширение:

```javascript
// background.js или content script
async function checkUrl(url) {
  try {
    const response = await fetch(
      `https://your-domain.com/api/lookup?url=${encodeURIComponent(url)}`
    );
    
    if (response.ok) {
      const data = await response.json();
      
      // Показать результат пользователю
      if (data.result.verdict === 'high') {
        showWarning(data.result.summary);
      }
      
      return data.result;
    } else if (response.status === 404) {
      // Домен не в кэше - выполнить полный анализ
      return await analyzeUrl(url);
    }
  } catch (error) {
    console.error('Ошибка проверки URL:', error);
  }
}

async function analyzeUrl(url) {
  const response = await fetch('https://your-domain.com/api/analyze', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ 
      input: url,
      telemetryConsent: true 
    })
  });
  
  return await response.json();
}
```

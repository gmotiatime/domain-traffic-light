# Доменный светофор - План реализации улучшений для конкурса

Этот документ описывает архитектуру и структуру данных для внедрения новых фичей:
1. Объяснение решений AI.
2. Система обучения на feedback.
3. Предсказание угроз.
4. Скрытая презентационная страница.

## 1. Объяснение решений AI

**Архитектура:**
- Обновлен промпт в `server/openrouter-proxy.mjs`. AI теперь получает четкую инструкцию возвращать 3-5 ключевых причин в виде структурированного списка с акцентом на причину и следствие ("что нашли + почему это важно").
- Точечная подсветка критичных сигналов.

**UI Компоненты:**
- В `AnalyzerPage.tsx` обновлен маппинг компонентов в блоках "Главный сигнал" и "AI-слой". Если `tone === "critical"`, блок перекрашивается в предупреждающие красные цвета (rose-500) с иконкой `ShieldAlert`.
- Язык адаптирован для подростков (избегание перегруженного технического жаргона).

## 2. Система обучения на feedback (Жалобы пользователей)

**Архитектура:**
- Эндпоинт `api/report.mjs` уже умел сохранять жалобы в Redis, привязывая их к конкретному хосту.
- Добавлена визуальная точка входа прямо из `AnalyzerPage.tsx` — кнопка "Неверный вердикт?", вызывающая `ReportModal`.

**Структура данных (Redis):**
- Ключ: `threat-cache:<version>:record:<id>`
- Модель `Report`:
  ```typescript
  {
    id: string;          // уникальный ID жалобы (timestamp + random)
    text: string;        // Текст пользователя
    verdict: string;     // Текущий вердикт ('low' | 'medium' | 'high')
    score: number;       // Оценка
    createdAt: number;   // Timestamp
    resolved: boolean;   // Статус обработки админом
  }
  ```

**UI Компоненты:**
- `AnalyzerPage.tsx`: Новая кнопка возле блока "Действие".
- `AdminPage.tsx`: Обновлена статистика, теперь админ видит общее количество жалоб и количество доменов, на которые поступили жалобы.

## 3. Предсказание угроз (Выявление новых паттернов)

**Архитектура:**
- Базируется на агрегации данных из системы жалоб.
- Фильтр "Подозрительные паттерны" в `AdminPage.tsx` автоматически ищет записи в кэше, у которых текущий вердикт "Безопасно" (`low`), но количество жалоб $\ge$ 3.
- Это сигнализирует модератору о том, что либо AI, либо локальный ruleset пропускают новый вид фишинга (например, новый популярный TLD для скама).

**UI Компоненты:**
- В `AdminPage.tsx` добавлен блок "Подозрительные паттерны" (отображается только если найдены такие домены).
- Показывает список доменов, количество жалоб, скор и TLD для быстрого визуального анализа.

## 4. Скрытая презентационная страница

**Архитектура:**
- Роут `#/presentation` добавлен в `src/lib/site-router.ts`.
- В `src/App.tsx` прописана логика рендеринга страницы без общих элементов интерфейса (Header/Footer с навигацией).

**UI Компоненты:**
- `PresentationPage.tsx`: Интерактивный компонент на базе `framer-motion` для плавной смены слайдов.
- Переключение слайдов с помощью стрелок на клавиатуре или кнопок управления.
- Слайды:
  1. **Intro**: Название проекта и цель.
  2. **Problem**: Зачем это нужно подросткам (рост фишинга).
  3. **Solution**: Как это работает (Ruleset + AI + Redis).
  4. **Demo**: Кнопка для запуска live-демо анализатора.
  5. **Future**: Планы по развитию.
